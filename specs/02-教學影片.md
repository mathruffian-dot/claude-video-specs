# 影片類型規範 #02：教學影片

> 給 agent 用的硬規範。看完應該能直接動工。
> 若使用者沒指定，預設依本文件執行；若衝突，以使用者明示為準。
> 同時參考 [01-活動紀錄影片.md](./01-活動紀錄影片.md) 中通用的渲染管線。

---

## 1. 定位

**用途**：把一個學科概念，用 3–8 分鐘的動畫影片完整講清楚，學生看完能複述、能應用。
**典型場景**：國小、國中、高中學科教學（數學優先），可延伸到自然、社會。
**核心三元素**：① **SOIL 教學脈絡**（先教學判斷再呈現）② **HyperFrames 視覺化動畫**（HF 的最大優點）③ **Edge-TTS 旁白**（免費、台灣腔）。

**和其他類型的差別**
- 不是活動紀錄片（不重現情緒，而是建構知識）
- 不是廣告片（無 CTA）
- **每頁有教學任務**，不是「漂亮的內容容器」
- **動畫服務理解**，不是炫技

## 2. 三層架構

```
┌─────────────────────────────────────────────┐
│ Layer 1：教學內容 — SOIL 六引擎               │
│   概念定位 → 脈絡 → 頁面 → 認知 → 風格 → 合成 │
├─────────────────────────────────────────────┤
│ Layer 2：視覺呈現 — HyperFrames + KaTeX     │
│   SVG 動畫、step-by-step 揭露、視覺化推理     │
├─────────────────────────────────────────────┤
│ Layer 3：聲音 — Edge-TTS                    │
│   zh-TW-YunJheNeural / HsiaoChenNeural      │
└─────────────────────────────────────────────┘
```

**順序不能反**：先想清楚教什麼（SOIL 1-3），再決定怎麼呈現（4-6），最後才處理聲音。

---

## 3. SOIL 工作流要點（必跑前三引擎再動工）

### 引擎 1｜概念定位稿
產出：總概念、3 子概念、3 常見誤解、帶走一句話、最小事實包、投影片 vs 旁白分配表。
**不能跳過**：直接做投影片會塞太多。

### 引擎 2｜脈絡定位稿
固定三段式：**引起動機（15–20%）→ 維持注意（60–70%）→ 喚起行動（15–20%）**。
每段標明：教學目的、入口、出口、建議頁數。

### 引擎 3｜頁面架構表
每頁四欄：**# / 時間 / 標題 / 頁面角色 / 唯一主重點 / 動畫描述 / ★關鍵**
頁面角色不超過 1/4 是同一種（避免單調）。

### 引擎 4-6 簡化版
- 4：每頁用六認知詞檢查（降雜訊、區塊化、增資訊、結構化、順脈絡、步驟化）
- 5：用本文件第 4 節風格規範（不用每次重新定）
- 6：合成 HTML（不是 pptx，因為我們要動畫）

## 4. 視覺設計系統（林長揚 30 原則 × SOIL 落地）

### 4.1 字體
**主字體**：**源石黑體 GenSekiGothic2TW**（粗圓感，符合林長揚 #7「字體選黑體粗體」）

| 字重 | 用途 |
|------|------|
| **H (900)** | 標題、強調數字、關鍵字 |
| **B (700)** | 副標、章節編號、字幕 |
| **M (500)** | 內文、註解 |

**嵌入方式**：
```css
@font-face {
  font-family: 'GenSeki TW';
  src: url('assets/fonts/GenSekiGothic2TW-H.otf') format('opentype');
  font-weight: 900;
  font-display: block;
}
/* 同樣方法載入 B (700) 與 M (500) */
```

> 字檔來源：`C:/Users/mathr/AppData/Local/Microsoft/Windows/Fonts/GenSekiGothic2TW-*.otf`。若新機器無字體：[ButTaiwan/genseki-font (GitHub)](https://github.com/ButTaiwan/genseki-font)。

### 4.2 字級階（林長揚 #1：55/34/21/13 黃金比例）
在 1920×1080 畫布同比放大：

| Token | 大小 | 用途 |
|-------|------|------|
| `--fs-hero` | **168–260px** | 封面、結語金句、巨大強調數字 |
| `--fs-title` | **120–140px** | 頁面主標題 |
| `--fs-sub` | **80–96px** | 副標題、章節標題 |
| `--fs-body` | **48–64px** | 內文、定義 |
| `--fs-note` | **24–32px** | 註解、章節編號、字幕 |

**行距 = 字大小的 50–75%**（line-height 1.15–1.45）

### 4.3 配色
| HEX | 名稱 | 用途 |
|------|------|------|
| `#FAF7EE` | paper（米黃）| 內容頁背景 |
| `#F2EDDC` | paper-2 | 卡片底色 |
| `#1A1A1A` | ink | 文字主色、封面/結語背景 |
| `#0E7C7B` | teal | **主強調色**（概念名詞、第一強調）|
| `#E36414` | coral | **副強調色**（重點數字、第二強調）|
| `#C8941F` | gold | 第三色，只用於微點綴 |

**林長揚 #25：強調色 1–2 種**。teal + coral 兩主色，gold 只在必要時用，不超過兩處。

### 4.4 版面（林長揚 #10「版面留白」反向應用）
**這類教學影片的留白要少**（與紀錄片相反）：
- padding：56px 80px 120px（上 / 左右 / 下）
- 內容區應**填滿至少 80% 畫布**
- 不要為了「美觀」留大片空白
- **林長揚 #14：元素對齊**：所有字卡、SVG 對齊基準線

### 4.5 必備 UI 元件
- **章節 chip**（左上）：`<span style="background:teal;color:white;padding:4px 12px">01</span> / 12 — 章節名`
- **進度條**（底部 5px teal）
- **字幕條**（底部，黑底白字，旁白逐句顯示）
- **黑色封面 + 黑色結語**（三明治結構，內容頁淺色）

## 5. 動畫規範（HyperFrames 的最大優點）

### 5.1 動畫類型
| 類型 | 用途 | 實作 |
|------|------|------|
| **逐元素揭露** | 配對找因數、列出質數 | CSS `transition-delay` 階層 |
| **路徑繪製** | 因數樹、知識地圖連線 | SVG `stroke-dasharray` 動畫 |
| **形狀重組** | 24 人分組（2×12 → 3×8 → 4×6） | JS 切換 grid-template-columns |
| **數字爆出** | 因數個數「6」、∞ | scale(0.5 → 1) + opacity |
| **數線標記** | 倍數逐點亮起 | 每個 tick 漸顯 |
| **網絡圖建構** | 知識地圖中心輻射 | 中心 → 分支線 → 節點 三步逐顯 |

### 5.2 動畫節奏準則
- 單一動畫 duration：**0.5–1.5s**
- 階層延遲：每階 **0.4–0.6s**（太快來不及看、太慢失節奏）
- **動畫服務理解**：每個動畫都要能回答「這個動畫，讓學生更懂了嗎？」
- 動畫結束後**靜止 1.5s 以上**讓學生消化

### 5.3 ★ 關鍵動畫（每支影片至少 2–3 個）
這是 HF 的招牌時刻，必須做好：
- **數學結構建構**：因數樹、Venn 圖、座標系
- **知識地圖**：總結頁的中心輻射，把各概念連起來
- **步驟拆解**：方程式逐步展開

## 6. 數學內容規範

### 6.1 方程式必須用 KaTeX
```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.10/dist/katex.min.css">
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.10/dist/katex.min.js"></script>
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.10/dist/contrib/auto-render.min.js"></script>
```
用 `$...$` 行內公式，渲染時呼叫 `renderMathInElement(slides[i], {delimiters:[{left:'$',right:'$',display:false}]})`。

### 6.2 數學圖形
- **幾何圖**：用 `jh-math-geometry` 技能（已內建於 SOIL skill）或自繪 SVG
- **座標圖、函數圖**：SVG 手繪或呼叫 geometry_renderer
- **嚴格要求**：圖形比例正確、標籤位置不重疊、刻度均等

### 6.3 數學正確性檢查（強制）
產出後**逐頁檢查**：
- [ ] 方程式 LaTeX 語法正確
- [ ] 數字計算正確（自行算一遍）
- [ ] 幾何圖形比例符合題意（如「等腰三角形」就要真的等腰）
- [ ] 函數圖切線、漸近線符合定義
- [ ] 沒有把「⋯」誤打成「...」（中數字符要對齊）

## 7. 旁白規範

### 7.1 TTS（免費）
**zh-TW-YunJheNeural**（少年男聲，預設）或 **zh-TW-HsiaoChenNeural**（女聲）
參數：`--rate -10% --pitch -2Hz`

### 7.2 文案
- 每頁 1–4 句，**讀完 8–25 秒**
- **第二人稱「你」** 較不適用，**第三人稱口語講解** 為主
- 重點概念用「我們說：⋯」「注意，⋯」「這就是⋯」當口語錨點
- 避免照念投影片文字，要**補充**而非**重複**

### 7.3 旁白生成腳本（範本）
```python
import asyncio, edge_tts
VOICE = "zh-TW-YunJheNeural"
async def synth(i, text):
    c = edge_tts.Communicate(text, VOICE, rate="-10%", pitch="-2Hz")
    await c.save(f"page-{i:02d}.mp3")

async def main():
    for i, t in SCRIPT:
        for r in range(3):
            try: await synth(i, t); break
            except: await asyncio.sleep(2)
```

> **重要**：必須**序列**執行，並行會被 Microsoft 端 reset connection。

### 7.4 字幕呈現
- 底部 30px 處，最大寬 88%
- `rgba(26,26,26,0.92)` 黑底白字，圓角 10px
- 字級 30px，font-weight 700（粗圓字仍清楚）
- 旁白播放時 `.show`，切頁時繼續顯示直到下頁字幕替代

## 8. 技術架構

### 8.1 專案結構（HyperFrames）
```
教學主題/
├── DESIGN.md                     # 套用本文件第 4 節即可，不必每次重寫
├── SCRIPT.md                     # 引擎 1-3 的產出 + 逐頁旁白
├── generate_narration.py         # Edge-TTS 序列生成腳本
├── assets/
│   ├── fonts/                    # GenSekiGothic2TW-H/B/M.otf
│   └── narration/                # page-01.mp3 ~ page-NN.mp3
├── renders/
│   ├── master_audio.mp3          # 對齊後的主音軌
│   └── 教學主題_v1.mp4
└── index.html                    # 主檔
```

### 8.2 主控制器（JS 範式）
```javascript
const PAGES = [
  { i:1, dur:14, sub:"字幕文字" }, // dur = 該頁總秒數（含旁白 + tail buffer）
  ...
];
function showSlide(i) {
  slides.forEach((s, idx) => s.classList.toggle('active', idx === i));
  sub.textContent = PAGES[i].sub;
  audio.src = `assets/narration/page-${pad(PAGES[i].i)}.mp3`;
  audio.play();
  // 觸發頁面特定動畫
  if (PAGES[i].i === N) playSlideN();
  // KaTeX 即時渲染
  renderMathInElement(slides[i], {delimiters:[{left:'$',right:'$',display:false}]});
}
```

### 8.3 渲染管線（與 #01 共用）
```bash
# 1. 生成所有頁的 padded audio
for i in 01..NN: ffmpeg -i page-${i}.mp3 -af apad -t ${dur} tmp/p${i}.mp3
# 2. concat 成主音軌
ffmpeg -f concat -i list.txt -c copy renders/master_audio.mp3
# 3. Playwright headless 錄 webm（在 GDrive 外執行）
node record.cjs  # 1920×1080, --mute-audio
# 4. mux
ffmpeg -i video.webm -i master_audio.mp3 -c:v libx264 -crf 20 -c:a aac final.mp4
```

> **GDrive 同步會破壞 node_modules**：Playwright 必須在 `C:/Users/mathr/AppData/Local/Temp/` 安裝執行。

## 9. Do / Don't

### ✅ Do
- 先跑 SOIL 引擎 1-3 才動工，不要直接寫投影片
- 每頁有明確的教學任務
- 字級用 5 層階級，標題粗、內文清楚分層
- 動畫服務理解（每個動畫都能說明為什麼這樣動）
- KaTeX 渲染數學，不用圖片
- 旁白與字幕分層、不重複
- 內容填滿至少 80% 畫布
- 強調色 teal + coral 為主、gold 為輔

### ❌ Don't
- 不要把講義照搬上投影片
- 不要為了「精緻」加 drop shadow、玻璃感、漸層裝飾
- 不要單一頁面角色超過 1/4
- 不要動畫太花俏（旋轉、彈跳過頭）
- 不要把所有內容塞同一張投影片
- **不要犯數學錯誤**（比例、符號、計算）
- 不要旁白照念投影片文字
- 不要混搭多種字體（一致用源石黑體）
- 不要在 GDrive 內裝 node_modules（會壞掉）

## 10. 給 agent 的執行 checklist

收到一份「教學影片」需求時，按此順序產出：

1. 確認 **學科 / 年級 / 單元 / 片長**（預設 4–6 分鐘）
2. 跑 **SOIL 引擎 1**：產出概念定位稿 → **給使用者確認**
3. 跑 **SOIL 引擎 2**：產出脈絡定位稿（三段式）→ **給使用者確認**
4. 跑 **SOIL 引擎 3**：產出頁面架構表（含★關鍵動畫標記）→ **給使用者確認**
5. 建立 HF 專案結構（複製 fonts 進 assets/）
6. 撰寫 SCRIPT.md（每頁旁白 + 字幕）
7. 跑 `generate_narration.py` 序列生成 mp3
8. 用 `ffprobe` 取得每頁旁白時長 → 加 2–4s tail 設為 page duration
9. 寫 `index.html`（依本文件第 4–6 節風格 + KaTeX + SVG 動畫）
10. **數學正確性自檢**（第 6.3 節清單）
11. 用 Playwright 在 temp 目錄錄製
12. ffmpeg mux → mp4
13. 給使用者預覽 → 確認後存檔
