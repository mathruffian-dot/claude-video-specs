# 影片類型規範 #03：社群科普影片

> 給 agent 用的硬規範。看完應該能直接動工。
> 若使用者沒指定，預設依本文件執行；若衝突，以使用者明示為準。
> 沿用 [02-教學影片.md](./02-教學影片.md) 大部分視覺基礎（字體、TTS、技術管線），只記錄差異點。

---

## 1. 定位

**用途**：把一個科普知識，用 2–3 分鐘濃縮成社群可分享的科普短片，由淺入深。
**典型場景**：科技 / AI / 科學 / 健康 / 金融 / 心理學 等科普主題；FB、IG Reels、YouTube Shorts、Threads。
**核心三元素**：① **強 Hook**（前 3 秒抓住滑手）② **多版面切換**（避免單調）③ **照片佐證**（科普概念需要視覺錨點）。

**和其他類型的差別**
- 不是教學片：受眾沒有「學會」的動機，要靠視覺與好奇心驅動
- 不是紀錄片：不重現情緒，重在解釋觀念
- **可靜音閱讀**：字幕必須自帶完整資訊（社群常靜音）
- **由淺入深強制**：先生活比喻 → 概念 → 機制 → 應用

---

## 2. 與 Type 2 的差異對照表（重要）

| 項目 | Type 2 教學 | **Type 3 科普** |
|------|------------|----------------|
| 片長 | 4–8 分鐘 | **2–3 分鐘** |
| 前 3 秒 | 可慢進入 | **強 Hook 必須** |
| 預設聲音 | 一定開 | **可能靜音** |
| 視覺密度 | 一頁一概念 | **一頁多視覺層** |
| 版面 | 偏固定 | **動態切換 ≥ 5 種** |
| 照片 | 偶用 | **常用** |
| 字幕字級 | 30px | **36px**（社群放大）|
| 進度條 | 5px | **8px** 漸層 |
| Hook 字級 | 不必特大 | **180–260px** |
| 結尾 | 帶走一句話 | **可選 CTA**「下次聊⋯」 |

## 3. 沿用 Type 2 的所有規範
- 字體：源石黑體 GenSekiGothic2TW H/B/M
- 旁白：Edge-TTS YunJheNeural（`rate -8% pitch -2Hz`，社群可比教學略快）
- KaTeX、SVG 工作流
- 渲染管線（Playwright + ffmpeg）
- 林長揚 30 原則
- 配色 teal `#0E7C7B` + coral `#E36414` + paper `#FAF7EE` + ink `#1A1A1A`

**新增配色**：`--neon: #FFD23F`（社群點睛色，只用在 Hook 強調字、進度條漸層末端、便利貼等情境）

---

## 4. 版面庫（必選 ≥ 5 種）

整支片**至少出現 5 種版面**，避免社群觀眾疲乏。

```
L1 全版照片        L2 左右並列        L3 圖示陣列(2x2/3x1)
┌────────────┐    ┌──────┬──────┐    ┌──┬──┬──┐
│ photo+mask │    │      │      │    │  │  │  │
│  大字 Hook  │    │ photo│ text │    │  │  │  │
│            │    │      │      │    │  │  │  │
└────────────┘    └──────┴──────┘    └──┴──┴──┘

L4 SVG 置中        L5 純大字          L6 對比並排
┌────────────┐    ┌────────────┐    ┌──────────┐
│   SVG 圖    │    │            │    │ A  →  B  │
│   diagram  │    │  HUGE TEXT │    │          │
│            │    │            │    │ before/  │
└────────────┘    └────────────┘    └──after───┘
```

### 各版面適用情境

| 版面 | 適合場景 | CSS 重點 |
|------|---------|---------|
| **L1 全版照片** | 過渡、章節分隔 | `background-size: cover` + 黑色 overlay `rgba(0,0,0,0.6)` |
| **L2 左右並列** | 概念導入（左照右字） | `display: grid; grid-template-columns: 1fr 1fr` |
| **L3 圖示陣列** | 列舉 N 點建議 | `grid-template-columns: repeat(N, 1fr)` |
| **L4 SVG 置中** | 機制圖、流程、節點圖 | `<svg viewBox>` + `.anim` 階段顯示 |
| **L5 純大字** | Hook、反轉、結語 | `background: var(--ink)` + 字級 180–260px |
| **L6 對比並排** | 前後對比、數字比較 | `grid-template-columns: 1fr 80px 1fr` |

### 版面切換準則
- **每相鄰兩頁不重複版面**（避免單調）
- 全片 12 頁建議：L5 × 2-3、L4 × 3-4、L2 × 2、L1 × 1、L3 × 1、L6 × 1
- **過渡頁用 L1**（全版照片 + 章節標題）
- **Hook / 反轉 / 結語必用 L5**（純黑大字最有衝擊力）

---

## 5. 敘事結構（由淺入深）

科普片有固定 5 段：

| 段 | 名稱 | 佔比 | 任務 | 推薦版面 |
|----|------|------|------|---------|
| 1 | **Hook** | 5–8% | 拋疑問或反常識 | L5 |
| 2 | **比喻** | 15% | 用生活物件類比 | L2 |
| 3 | **機制** | 35% | 拆解原理（含 SVG）| L4 × 多 |
| 4 | **反轉 / 應用** | 30% | 「不是 X，是 Y」+ 3 個招式 | L5 + L6 + L4 |
| 5 | **結語** | 10% | 一句金句 | L5 |

## 6. 素材策略（重要新章節）

### 6.1 三層備援

| 層 | 來源 | 需要 | 適合場景 |
|----|------|------|---------|
| **Tier 1 · 通用** | **Unsplash 直連** | 無 API key | 99% agent 都能用，預設 |
| Tier 2 · 進階 | Pexels / Pixabay API | 免費註冊 | 找不到合適 Unsplash 時 |
| Tier 3 · 客製 | draw 技能（gpt-image-2）| OpenAI 驗證 + 付費 | **選配**，找不到現成圖才用 |
| Tier 4 · 影片素材 | Pexels Videos / Coverr | 多數 CC0 | 動態素材需求 |

### 6.2 Unsplash 直連工作流（給其他 agent 用）

```bash
# 1. 用 WebFetch 搜尋特定關鍵字
WebFetch("https://unsplash.com/s/photos/<keyword>", "List top 5 photo IDs")

# 2. 取得某張照片完整 ID
WebFetch("https://unsplash.com/photos/<photo_id_short>", "Direct download URL")

# 3. 下載到專案
curl -sL -o assets/images/<name>.jpg \
  "https://images.unsplash.com/photo-<full-id>?w=1920&auto=format&fit=crop&q=80"
```

**注意**：
- `source.unsplash.com` 已於 2024 年棄用，**請使用 `images.unsplash.com/photo-XXX`**
- 圖片**下載到本地**，不要外部依賴（其他 agent 複製專案就能跑）
- 授權標示寫在頁尾或 README

### 6.3 照片使用準則

| 用法 | 處理 |
|------|------|
| **L1 全版背景** | `rgba(0,0,0,0.5–0.7)` 暗化 overlay 讓字讀得清 |
| **L2 左欄照片** | 不暗化，但用 `linear-gradient` 在右側淡出接文字欄 |
| **連貫性** | 同主題 2–3 張照片色調盡量一致 |
| **裁切** | 直式照片用 `object-fit: cover` + `object-position: center` |

### 6.4 何時用 draw 技能（選配）
- 需要**特定情境**且 Unsplash 找不到（如「AI 機器人在課堂講課」）
- 需要**多張一致風格**（draw 支援多圖一致性）
- 用前先問使用者是否願意付費

## 7. SVG 動畫範式

科普片的 SVG 動畫**比教學片更重要**（因為觀眾注意力短）。

### 7.1 標準動畫類別
```css
.anim { opacity: 0; transition: opacity 0.6s ease, transform 0.6s ease; }
.anim.in { opacity: 1; }
```
```javascript
function triggerAnims(slideEl) {
  slideEl.querySelectorAll('.anim').forEach(el => {
    const d = parseInt(el.dataset.d || '0', 10);
    setTimeout(() => el.classList.add('in'), d);
  });
}
```

每個 SVG 元素加 `class="anim" data-d="800"`（毫秒延遲），自動階段顯示。

### 7.2 推薦的科普 SVG 模式
- **節點 + 連線**：拓樸圖、概念關係
- **逐步堆疊**：對話貼上便利貼、token 切分
- **箭頭溢出**：擠出、流動
- **大數字對比**：5000 → 500（用 L6）
- **網絡放射**：主 → 多分支

## 8. 字幕特化（社群關鍵）

```css
.subtitle {
  position: absolute; left: 50%; bottom: 50px; transform: translateX(-50%);
  max-width: 90%; padding: 18px 40px; border-radius: 12px;
  background: rgba(26,26,26,0.92); color: #fff;
  font-weight: 700; font-size: 36px; line-height: 1.4;  /* ← 比 Type 2 大 6px */
  text-align: center;
}
```

**字幕原則**
- 字幕**獨立成段、自帶完整資訊**（不依賴聲音）
- 每段不超過 25 字（社群一眼讀完）
- 保留旁白原文，但可加 emoji 強化視覺
- 旁白可比字幕略長（補充說明）

---

## 9. 進度條漸層

```css
.progress {
  height: 8px;
  background: linear-gradient(90deg, var(--teal), var(--neon));
}
```

社群觀眾在意「還剩多久」，進度條要顯眼。漸層 teal → neon 讓終點更亮。

---

## 10. Do / Don't

### ✅ Do
- **前 3 秒下 Hook**（疑問句或反常識）
- 至少 5 種版面交替
- 字幕加大 36px、永遠在
- 至少 1 張背景照片（L1 或 L2）
- 結尾 L5 + 金句
- 用 Unsplash 為主、draw 為輔
- 進度條加 neon 漸層

### ❌ Don't
- 不要超過 3 分鐘（社群會滑走）
- 不要連續 2 頁同版面
- 不要全片無照片（純圖表會像簡報而非短影音）
- 不要 Hook 太溫和（要「為什麼⋯」「不是⋯而是⋯」）
- 不要假設觀眾開聲音
- 不要強推 draw 技能（不一定每個 agent 都能用）

## 11. 給 agent 的執行 checklist

1. 確認 **主題 / 受眾 / 平台**（預設 FB / YT 16:9，可選 9:16）
2. 撰寫 **5 段式腳本**（Hook → 比喻 → 機制 → 反轉 → 結語）
3. 每頁標明 **版面類型 L1–L6**，確認 ≥ 5 種
4. 列 **照片需求清單**，優先 Unsplash 直連
5. 用 WebFetch 找照片 ID → curl 下載到 `assets/images/`
6. 建立 HF 專案結構（複製 fonts）
7. 撰寫 SCRIPT.md（旁白 + 字幕 + 動畫描述）
8. 跑 `generate_narration.py` 序列 Edge-TTS
9. 寫 `index.html`（套 6 種 layout class + `.anim data-d`）
10. 用 Playwright 在 temp 目錄錄製
11. ffmpeg mux 主音軌 → mp4
12. 給使用者預覽
